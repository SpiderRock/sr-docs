Risk Differences - V7 to V8

Accessing the Risk Controls​

The Control Viewer (CV) used in Version 7 (V7) for viewing and editing risk controls has been deprecated and replaced by the Portal Application in Version 8 (V8).

To access the risk controls in Portal:

  1. Log in to SpiderRock Connect using either:

    • The web: https://spiderrockconnect.com/portal
    • Or the installed desktop application
  2. Once logged in, open the Portal Application.
  3. Click the Portal Icon to reveal the search bar.
  4. In the search bar, type "Risk Control". This will display the available types of risk controls that users can choose from:

For this document, we will focus on MarRiskControl, SpdrRiskControl (previously Supervisory Risk Control in V7), and MarRiskControlTkOverride, which can all be edited by Firm Risk Admins.

Market Access Risk Controls (MAR)​

Market access risk refers to the potential financial and regulatory exposures that arise when broker-dealers provide clients with direct access to financial markets. To address these risks, SpiderRock implements MAR Risk Controls that provide pre-trade validation, real-time monitoring, and multi-layered protection systems. These controls ensure compliance with SEC requirements while protecting both SpiderRock and its clients from excessive market exposure, allowing for safe and efficient market access with comprehensive risk management. For more information about MAR risk controls, please visit the Execution Risk Control page or the Integrated MAR Risk Controls page on this site.

MarRiskControl in V7​

In V7, the Market Access Risk tab is only visible to Client Risk Admins. The limits are global and are applied at the risk group level.

Under the MarRiskControl tab, double-clicking on any of the values in the grid or clicking on the "Add New" button (top-right corner) will bring the Order/Rules Limits window displayed in the inset. Authorized users can edit these rules and limits and post their changes by clicking on the box in the bottom-right corner of the window (see green check mark).

MarRiskControl in V8​

In V8, MarRiskControl records are used to establish risk controls within SpiderRock's execution engines. These records can only be edited by users who have been given explicit permissions by SpiderRock. Some MAR limits in V8 have been slightly modified from V7 and are explained in further detail below.
Clicking on the MarRiskControl chip filter (red area circled below) will display all existing records in a grid format. To create a new MarRiskControl record, users must click on the plus symbol in the top right corner of the Portal and select "Add New Record" (see red arrow below).

This opens the MarRiskControl panel where users can set specific limits for different asset types. At the top of the panel (see the red area below), the user can input:

  • Risk Firm: If it exists, it will be the client firm that is imposing the risk controls (It must be an existing SpiderRock client firm).
  • Risk Group: ID of the risk group these rules will apply to (It will usually be the client firm value unless a custom risk group has been applied).
  • Test Risk Group: If it is a test risk group, this control applies only to test accounts.

MAR Risk Limits​

Within the MarRiskControl panel, users can expand or collapse multiple dropdown menus, each containing customizable features and limits to manage trading risk across asset types. The categories are:

1. Portfolio Risk Limits​

These controls apply to portfolio-level exposure (see yellow areas outlined above).

  • Margin Limit Account: Maximum net portfolio margin (start-of-day positions + day trades).
  • Margin Limit Day: Maximum net day-only portfolio margin (day trades only).
  • Open Exposure Limit:
    • Maximum absolute dollar delta of open child orders (no netting).
    • Scope: Applies to open child orders only; evaluates directional exposure. This differs from V7, where we apply a Total Exposure Limit, which was Margin + Open Orders.

2. Asset Class Enabled​

In V8, users can enable or disable trading by asset class (stocks, options, and futures) using a toggle switch (outlined in blue below).

3. Order Quantity​

The maximum number of shares, option contracts, or futures contracts that a single parent order can buy or sell. See the red outlined areas below.

4. Order Dollar Delta​

This is the maximum amount of dollar delta for a single parent order for its asset (see the yellow areas outlined below). This feature is commonly used for setting risk thresholds to control exposure per parent order and for performing limit checks to ensure no single order creates excessive market risk. In V7, most of the order limits were expressed as notional values. In V8, the dollar delta limits are more prevalent.

The following table shows the formulas used to calculate dollar deltas for different asset types:

Asset TypeFormula
Stocksshares * pointValue * midPrc
Optionscontracts * pointValue * ABS(de) * uMidPrc
Futurescontracts * pointValue * midPrc

5. The Collar Percentage​

The collar percentage is a risk control parameter used to restrict order prices by defining a maximum allowable deviation between a user's submitted price and the current bid/ask prices in the market. It is expressed as a decimal number between 0 and 1 and is entirely new to Platform V8. See the green areas outlined above.

Practical Example: The system rejects orders if the submitted price deviates from the bid/ask by more than the specified collar percentage. Let's assume the collar percentage is set to 0.05 (i.e., 5%).

Then:

  • Buy order
    If the best ask price for a stock is $100.00, the highest allowable buy price is:
    100 + (100 * 0.05) = $105.00
    Any buy order > $105.00 would be rejected since it is outside of the collar.

  • Sell order
    If the best bid is $100.00, the lowest allowable sell price is:
    100 - (100 * 0.05) = $95.00
    Any sell order < $95.00 would be rejected as outside the collar.

Specific Risk Limits for Futures​

The futures dropdown menu in the MarRiskControl panel has additional fields specific to futures (see the red areas outlined below).

Below is a detailed explanation of each field:

FieldDefinition
maxAccFutCnAbsThe maximum number of futures contracts the account can hold at any one time, counting both long and short positions, as a net value (includes start-of-day positions).
maxDayFutCnBotThe maximum number of futures contracts that can be bought during the trading day.

Order Controls​

Certain market orders can be configured using the order controls dropdown menu (see the red area outlined below). Only SpiderRock admins can edit these controls.

Below is a detailed explanation of each field:

ControlDescription
Allow Stk Odd LotsAllows parent stock orders with an order size below 100 shares.
Allow Market on Close (MOC)Enables submission of MOC orders, which are executed as close as possible to the market close at the final trading price of the day.
Allow Limit on Close (LOC)Similar to MOC but includes a price limit. The order will only execute at the closing price if it is at or better than the specified limit.
Allow ISO Sweep OrdersEnables Intermarket Sweep Orders (ISO): market orders that allow traders to bypass the Order Protection Rule (Rule 611) and access liquidity on other exchanges immediately.
Block Short Sales (Block SSales)Completely blocks any short sale orders, regardless of asset type.

SpdrRiskControl (Supervisory Risk Controls)​

SpdrRiskControls, previously referred to as supervisory risk controls in V7, are tactical controls that allow the user to set tighter restrictions at the account, symbol, and user levels. They can constrain child orders and, when needed, reject parent orders to keep risk within defined parameters. For more information about these controls, please visit the Execution Risk Controls page or the Integrated Supervisory Risk Controls page on this site.

Supervisory Risk controls in V7​

In V7, supervisory risk controls can be found under the Supervisory Risk tab in the Control Viewer. This tab allows the user to set risk controls at the symbol, user, or user/symbol level.

Double-clicking any value in the grid or clicking on the “Add New” button (top-right corner) will open the Order Limits window displayed above. Users can edit their order and day limits by clicking the corresponding tabs. To save, users select Post Changes in the lower right-hand corner of the window.

SpdrRiskControl in V8​

In V8, the supervisory risk controls have been condensed into the SpdrRiskControl table within the Portal Application.

After clicking the SpdrRiskControl chip filter (red area circled below), existing records will be visible in a grid format. To create a new SpdrRiskControl record, users must click on the plus symbol in the top right corner of the Portal application and select "Add New Record" (see red arrow below).

This opens the control panel where users can edit certain limits and apply various controls at the client firm, symbol, user, and account levels. At the top of the panel (see the red outline below), the user can input:

  • Ticker: Must be formatted as TK-TS-AT (Ticker Symbol; Ticker Source; Asset Type)
FieldDefinitionValues
Ticker SymbolUnique series of letters or characters used to identify a publicly traded security on a specific exchange (e.g., TSLA for Tesla).'*-ANY-ANY' is the default ticker symbol (if a more precise ticker control has not been set). It applies all the user’s limits to every symbol for a specific account.
Ticker SourceOrigin or provider of the underlying market data used to price, quote, and manage trades for a given security or derivative.NMS, CME, CBOT, COMEX, NYMEX, CFE (All listed US equity, equity option, and index options will fall under the ticker source NMS).
Asset TypeUnderlying financial instrument that the option contract is based on.EQT, FUT
  • Risk Control Level: User has the choice to set this at the clientFirm, account, user, or user/account level.

  • Risk Control Key: Depends upon which risk control level is selected.

    • If set at the clientFirm level, the risk control key is ignored.
    • If set at the account level, the risk control key must be a SpiderRock client account (e.g., t.test).
    • If set at the user level, the risk control key must be a SpiderRock user (e.g., cf.username).
    • If set at the user/account level, the risk control key must be a SpiderRock user and a SpiderRock client account separated by a colon (e.g., cf.username:t.test).
  • Risk Firm: Client firm imposing risk controls (It should be an existing SpiderRock Client Firm).
  • Is Test Account: Determines whether the account is a test account or not.

Control Panel​

Users can enable or disable stocks, futures, and options within the Control Panel (see the yellow area outlined above). The CloseOnly setting allows users to close out existing positions. For example, if you are long 5,000 shares of AAPL, you can sell up to 5,000 shares to close the position, but you cannot buy any additional shares. This functionality replaces the MarRestrictedList feature from V7.

Day and Exposure Limits​

To configure daily trading risk thresholds, users can set limits in the day limit panel. These limits apply to each trading session and are monitored in real-time to manage exposure across accounts (see the red outlined area below).

Users can also enter an Open Exposure Limit in the exposure limit panel (see the yellow area outlined above). This limit caps the total dollar exposure from all pending orders, regardless of whether they're long or short positions. The values are cumulative and not netted across a symbol. 

Below is a detailed explanation of each field in the day and exposure limit panels:

Field NameDefinition
marginLmtDayMaximum net (per symbol/day) day portfolio (day trades only) margin (can include external sources)
maxDayDdLnOpen long dollar delta per trading session
maxDayDdShOpen short dollar delta per trading session
maxDayDdAbsAbsolute dollar delta, |ddBot| + |ddSld| position per trading session
maxDayWtVeLnOpen long WtVega (weighted time vega) per trading session
maxDayWtVeShOpen short WtVega per trading session
maxDayWtVeAbsOpen absolute WtVega, |WtVeBot| + |WtVeSld| per trading session
maxDayNvalueLnOpen long notional value per trading session
maxDayNvalueShOpen short notional value per trading session
maxDayNvalueAbsAbsolute notional value, |NValueBot| + |NValueSld| per trading session
opnExposureLmtMaximum abs open child order $Delta (no netting) (open child orders only)

Where WtVega = Weighted Time Vega (time-decayed measure of volatility exposure). WtVega considers both the vega of an option and its time to expiration, weighted accordingly.

MarRiskControlTkOverride​

In V8, we’ve introduced the ability to add symbol-specific limits at the MAR level. MarRiskControlTkOverride records are used to establish ticker-specific risk controls within SpiderRock's execution engines. These records are only viewable and editable by users belonging to the same risk firm as the control records.

After clicking on the MarRiskControlTkOverride chip filter (red area circled below), existing records will be visible in a grid format. To create a new MarRiskControlTkOverride record, users must click on the plus symbol in the top right corner of the Portal application and select "Add New Record" (see red arrow below).

This opens the control panel, which allows users to configure nearly all the same risk parameters and limits available in MarRiskControl, with the added functionality of setting symbol-specific limits. At the top of the panel (outlined in red below), users can input the following fields:

  • Risk Firm
  • Risk Group
  • Test Account Indicator (Yes/No)
  • Ticker

The ticker field must be formatted as TK-TS-AT (Ticker Symbol; Ticker Source; Asset Type).

Control Panel​

Users can enable or disable stocks, futures, and options in the control panel (see the yellow area outlined below). They can also block short sales.

Order Limit Panel​

Users can set the order quantity, dollar delta, and collar percentage for stocks, futures, and options in this panel (red area outlined below). For detailed definitions of these fields, please refer to the MAR Risk Limits section of this guide.

Futures Position Limit​

The user can set these specific position limits for futures (see the yellow area above):

FieldDefinition

maxDayFcBotMaximum number of futures contracts the account is allowed to buy during the trading day.
maxDayFcSldMaximum number of futures contracts the account can sell during the trading day.
maxDayFcAbsMaximum net total number of futures contracts that the account can trade during the day (buying and selling combined).

The user can publish the record by clicking on the blue button in the bottom right corner of the panel (see the red arrow above).

MarRiskComposite​

The MarRiskComposite feature is entirely new to V8. It combines the MarRiskControl and the MarRiskCounter, allowing the user to easily view their MAR risk limits and counters all in one place. This table is NOT editable; it is for viewing only. To access this feature, click the portal icon in the top-left corner of the application, type "MarRiskComposite" in the search bar, and select it.

Accessing MarRiskComposite Records​

To view existing records, users can click the MarRiskComposite chip filter (see the red circle below). This action will display all available records in a grid view. From the grid, selecting any record will open the control panel, where all configuration details can be viewed.

Users can view but not modify the ticker, risk group, risk firm, and test account toggle as well as the portfolio risk limits.
