Getting Started

Introducing SpiderRock Connect​

Welcome to SpiderRock Connect (also known as Version 8 or V8), the next-generation SpiderRock trading platform. SpiderRock Connect introduces a range of new capabilities while continuing to support legacy Citrix tools and an enhanced version of the SRSE API. The new features include reimagined web-application tools, our new MLink API, and the SpiderRock ATS, which enables RFQ-style trading for listed and FLEX options. If you are a new client, please contact our Sales department at learn@spiderrock.net to discuss how to get started with SpiderRock Connect.

    If you are an existing client that has already been onboarded onto SpiderRock Connect and you have questions about obtaining credentials to a specific API, please reach out to our Client Support Desk at cs@spiderrock.net.

Accessing GUI Tools​

Legacy Tools / Citrix Workspace​

Users can access SpiderRock's legacy tools through the Citrix Workspace:

  1. The SpiderRock Client Support Desk will generate a password for the user (sent via secure PDF)
  2. Users can log in at http://spiderrock.cloud.com

Next-Generation Tools / WebApp Tool Suite​

Users can get access to the full WebApp Tool Suite by downloading the SpiderRock Connect desktop application here:


Getting Started with WebApp Tools​

  1. Contact the Client Support Desk to create your SpiderRock ID — an invitation will be sent to your email.
  2. Follow the link in the email to complete registration.
  3. At the login prompt, enter your password that was established with SpiderRock during the onboarding process. The username field should already be filled with the username associated with your user in SpiderRock's system.
  4. Complete MFA verification (authenticator app recommended).
  5. You'll land on the new Launcher browser page, where you can open any tools you're configured to access.
note

For additional assistance with login information, download issues, or the Tool Suite installer, contact the SpiderRock Client Support Desk at cs@spiderrock.net or your IT department, depending on the nature of the issue.
