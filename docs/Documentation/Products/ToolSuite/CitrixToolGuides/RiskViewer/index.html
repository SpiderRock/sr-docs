Risk Viewer

The SpiderRock Risk Viewer is a real-time portfolio risk management tool allowing users to view and aggregate risk at the account level. A portfolio can contain any combination of equity, options, and futures positions of any size. The tool is organized in tabs so that users have quick access to position risk, PnL allocation, and market shock analysis. The tool also offers guidance around PIN risk, early exercise opportunities and provides visibility on position breaks.

The Risk Viewer Tabs​

The table below briefly describes all Risk Viewer tabs. Each tab comes with a default view. The user can customize these views by right-clicking on headers and selecting the desired metrics.

Position Risk Tab​

Displays real-time position risk across Greeks at the root symbol level (across asset classes).

  • User has access to most Greeks associated with symbol position and skew buckets within a position: delta, delta,delta, delta,beta, gamma, $gamma, theta, weighted Vega, weighted time Vega, absolute Vega, rho, as well as others
  • Other metrics include an average implied volatility move overnight and pre-set risk slides
  • Other metrics can be selected by right-clicking on any column headers

Position PnL Tab​

Displays overnight portfolio position PnL and PnL allocation across Greeks at the root symbol level

Many PnL allocations are available across Greeks (pnl.ga, pnl.de, pnl.th, pnl.ve), PnL from trades today (day.pnl), and hypothetical day PnL based on positions being hedged every minute (hdg.pnl).

The system also provides three distinct PnLs from start-of-day positions:

  • clr.pnl: PnL from cleared marks on the open
  • mid.pnl: PnL from the SpiderRock mid NBBO marks (snapshot taken 30 seconds before close)
  • vol.pnl: PnL from the SpiderRock volatility surface marks from the previous EOD
  • Other metrics can be selected by right-clicking on any column headers

Position Detail Tab​

Client-centric industry-based information are associated with each symbol position.

  • $amt: Dollar amount of the next expected dividend
  • d.dy: Days to next dividend ex-date
  • edays: Days to next expected earnings. A negative number indicates days from a recent announcement
  • stk.volume/opt.volume: Trailing 30-day average daily volume for stocks/options
  • locate: Available locate quantity per symbol
  • Other metrics can be selected by right-clicking on any column headers

Position VaR Tab​

Analyzes PnL impact of pre-canned risk slides for each symbol position across the portfolio.

  • Haircut25/haircut37 provides a real-time aggregate margin consumption per symbol
  • Other metrics available are pre-set risk slides such as: up and down 5%, 15%, 50% (volatility unchanged), earning volatility move (underlying price unchanged). More pre-set metrics can be selected by right-clicking on any column headers
  • Other metrics can be selected by right-clicking on any column headers

Industry Risk / Expiration Risk Tabs​

Displays aggregate position risk in real-time across industry sectors and expiration buckets.

  • abs.cur.cn is the position in absolute number of contracts based on the aggregation chosen
  • Greek ratios (Long - Short over Abs) for Vega, Weighted Vega, Weighted-time Vega, and Theta
  • Aggregate time-weighted vega by skew bucket (wtVe.dn/downside, wtVe.at/at-the-money, wtVe.up/upside)
  • Net Vega (nVega), net weighted Vega (nWvega), net theta (nTheta)
  • Ratio of Long Wvega over Long Vega (ivolLn), ratio of Short Wvega over Short Vega (ivolSh)
  • Aggregate premium over parity (prem.opar)
  • Aggregate dollar delta ($delta)
  • Other metrics can be selected by right-clicking on any column headers
      Note: Users can select Industry, Sub-Industry, or Group as a filter under the Industry Risk Tab.

Theo Concentration Tab​

Displays theoretical edge (and other values) for clients trading based on their own theoretical volatility model.

  • avg.iv: average implied volatility for the position in a particular symbol
  • opt.edge.opened: concentration in edge for open positions
  • opt.edge.closed: concentration in edge for closing positions
  • t.edge: aggregate dollar theoretical edge based on user's theoretical values
  • unit.edge: theoretical edge per contract (normalized to a $40 stock)
  • Other metrics can be selected by right-clicking on any column headers

Theo Results Tab​

Displays theoretical edge metrics from the previous day for clients trading based on their own theoretical volatility model.

  • t.edge.pr: prior day aggregate dollar theoretical edge based on user's theoretical values
  • u.edge.pr: prior day theoretical edge per contract (normalized to a $40 stock)
  • tv.chg: average change of theoretical vol from previous day
  • iv.chg: average change of implied vol from previous day
  • Other metrics can be selected by right-clicking on any column headers

Future Risk Tab​

Displays intra-day position risk in real-time for futures positions at the symbol level.

  • uprc: spot price
  • avg.iv: Symbol average implied volatility
  • All Greeks associated to symbol position
  • Weighted-time Vega by delta buckets (at-the-money, downside, and upside buckets)
  • Other metrics can be selected by right-clicking on any column headers

Future PnL Tab​

Displays overnight portfolio futures PnLs at the symbol level and allocates PnL across Greeks.

Many PnL allocations are available across Greeks (pnl.ga, pnl.de, pnl.th, pnl.ve), PnL from trades today (day.pnl) and hypothetical day PnL based on positions being hedged every minute (hdg.pnl)

The system also provides three distinct PnL from start-of-day positions:

  • clr.pnl: PnL from cleared marks on the open
  • mid.pnl: PnL from the SpiderRock mid NBBO marks (snapshot taken 30 seconds before close)
  • vol.pnl: PnL from the SpiderRock volatility surface marks from the previous day
  • Other metrics can be selected by right-clicking on any column headers

Pin Tool Tab​

Lists all positions with potential pin risk on expiration day.

This tool displays the minimum pin distance for the trade as a component of a symbol position. The user can use the '+ sign' on the left of the account column to expand the position row and see all the trades that comprise the position. The user can then see the pin distance (pinDist) which is the spot price-strike price for each trade

In the upper right corner of the tab, the user can enter a Pin Band (see below). Based on this reference number, the trades composing the position might return an action if within that Pin Band. Also, the reference can be either the current underlying price (uPrc) or the underlying price of the last print (uPrt)

Early Ex Tab​

Lists potential early exercise or assignment candidates.

This tab displays all trades that can be potentially exercised early or assigned. The risk alert column provides an indication as to what action may exist with regard to a potential trade. The risk alerts are:

  • ExAskSoon: Client might exercise option early based on ask price
  • ExAskNow: Client should exercise option early based on ask price
  • ExSurfSoon: Client might exercise option early based on surface price
  • ExSurfNow: Client should exercise option early based on surface price
  • ExBidSoon: Client might exercise option early based on bid price
  • ExBidNow: Client should exercise option early based on bid price

Calls​

For calls, the test for early exercise is triggered when the next dividend payment is scheduled to occur in the next 5 days. The exercise condition is approximated by comparing the value of the next dividend with the value of the corresponding put (same strike and expiry). If the value of the dividend is larger than the price of the put, a riskAlert value will be assigned, and that option row will be displayed in the Early Ex tab.

The riskAlert value is assigned according to two criteria: the highest price level for which early exercise is triggered, and the proximity of the ex-div date. For price level, the next dividend is compared against the put ask price, then (if the ask is larger than the dividend) the SpiderRock surface price, and finally the put bid price. If the dividend goes ex tomorrow, the riskAlert will indicate “Now”, otherwise it will indicate “Soon”. For example, ExBidSoon indicates that the stock will go exdiv but within 5 days but not tomorrow, and the SpiderRock put surface price is greater than the next dividend, while the bid price is less than the dividend.

The ex.value is reported as the difference between the next dividend and the SpiderRock surface put price.

    Note that if the riskAlert is only triggered at the Bid level, then the ex.value for the call wil be negative; otherwise, it will be positive.

Puts​

For puts, the test for early exercise is always checked. The exercise condition is approximated by comparing the carry on the strike, defined as strike×(exp((rate−sdiv)×years))strike \times (exp((rate-sdiv) \times years))strike×(exp((rate−sdiv)×years)) with the value of the corresponding call plus the accrued dividends to expiration (the ddiv value).

Similar logic is used in assigning the riskAlert value for puts, and the ex.value is reported as the difference between the strike carry and the SpiderRock surface call price plus any accrued dividends to expiration.

General Settings and Filters​

Aggregation Row​

Each tab has an aggregation row above the grid, where certain metrics are aggregated across the entire portfolio.

Column Properties​

By default, a certain set of columns and associated colors are displayed for each tab. However, views can be customized to suit individual requirements by using the column properties window, where the user can select/deselect values, modify fonts and color schemes.

Column Properties Window​

  • Caption: The ‘Caption’ section shows the name of the column
  • Show: To choose whether a column is displayed, check or uncheck the box in the ‘Show’ section. If you right-click anywhere in this window, you may choose to quickly select all or none of the available columns
  • Filter: To view or apply filters for a given column, left-click the blue funnel symbol in the ‘Filter’ section. This will open a pop-up menu allowing you to select the criteria by which to filter
  • Colors: The ‘Colors’ section allows you to select a background and foreground color for that column, to aid in quick identification
  • Description: The ‘Description’ section provides a short explanation of what is tracked by the column in question
  • Group: The ‘Group’ section bundles columns by related subject matter. Clicking ‘Group’ or ‘Caption’ will allow the user to quickly sort this window alphabetically by the relevant option
  • Search: The ‘Search’ box allows the user to quickly find the desired column without scrolling through an extensive list. The ‘Group’, ‘Caption’ and ‘Description’ boxes to the right allow the user to select in which sections to search for the word or phrase entered
  • Width: The (+) and (-) buttons allow the user to adjust the width of columns in Risk Viewer by increments. The Auto Width button will resize columns to the smallest width required to display all data
