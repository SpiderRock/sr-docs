Order Manager V2

The new SpiderRock Order Manager - Order Manager V2 - is a live order-and-execution management tool, allowing users to monitor intra-day trading activity at the portfolio level. The user can filter and aggregate metrics for both working and filled orders, manage legged orders, the status of any auto-hedging activity as well as rejects. The Order Manager V2 is a convenient tool to mass cancel orders. Please note that this tool is a work in progress, and more functionalities such as sounds, and editing/triggering orders will be added in the near future.

General Functionalities Across Tabs​

The Order Manager V2 launches on the Executions tab and provides a default view displaying all accounts enabled for the user. The tool is currently organized into 5 tabs: Executions, Orders, MultiLeg Orders, AutoHedge States and Rejects. This document will highlight the main functionalities for each tab.

All tabs display the following icons in the top right corner, allowing the user to filter, create custom views, select accounts, cancel orders and monitor alerts.

Smart Filtering​

The user can filter by using the lookup box in the upper right corner of the tool as displayed below.

Users can use a wildcard which is a star: _ before or after typing in text. For example, typing T.CH_ would display results for all T.CH iterations such as T.CH, T.CH1 and T.CHA. To filter by more than one component, users can leave a space between each field. For example, T.CH TSLA would display all TSLA activity in the account T.CH. Users can use a NOT operator by typing ^. For example, ^.T.CB would display all accounts except T.CB. Note, the NOT filter works in combination with the wildcard filter.

Another way to filter is to right-click on any values in the grid, select ‘Filter on’ and select Account, RiskGroupID, RiskHash, Ticker or Username.

Creating Up to 6 Different Views​

Views can be created via the column selector described below. The views can be recalled by clicking on the number selector or using the hotkeys Ctrl + 1 to 6. Users have up to 6 custom views. After selecting the metrics, users can rearrange columns which will be saved automatically under the current view.

Column Selector​

The window below is obtained by right-clicking on the column header and selecting the column chooser. The available columns list is displayed to the left. The visible columns list is displayed to the right. Users can highlight from the available column list and drag to the visible column list to create a view. Views are selected by clicking on one of the numbers in the red box.

Visible columns can be rearranged using the 4 arrows above the list. The first up arrow moves the current selected row above by one. The next up arrow with a bar moves the current selected row to the top of the list. The down arrow moves the current selected row down by one. The last down arrow with the bar moves the current selected row to the bottom of the list.

Users have access to a search dialog box to find specific metrics. The window provides a comprehensive description for each metric. The search engine allows users to search across the Caption, Group and Description fields. Toward the bottom of the window, users can customize the display of the grid by selecting a density, color scheme and format.

Account Selector​

The window below is obtained by clicking on the silhouette icon. By default, users see all assigned accounts when opening the tool. However, users can select/deselect accounts by checking/unchecking the square boxes to the left of the account list. Users can also filter by typing an account in the filter box above the list. Lastly, users can select/deselect all accounts using the buttons at the bottom of the list.

Cancel Orders​

Users can click on the cancel widget (red arrow in the screenshot below) to cancel orders across select accounts. When the widget is clicked, a pop-up window will appear, displaying all accounts the user has access to. Users have the option to mass cancel active orders across all accounts by clicking on the red button ‘Cancel Orders For All Accounts’. Alternatively, they can cancel all orders in an individual account by clicking on the red X button to the right of the desired account.

Notifications​

The bell in the upper right corner is an alert feature. This is currently a work in progress and once completed, it will turn red if any notifications have been sent by the support desk.

Executions Tab​

  • The Executions Tab shows executions for all orders in the SpiderRock system regardless of source.
  • The Executions Tab offers an aggregation functionality allowing for three aggregation buckets to view fills.

Orders Tab​

  • The Orders tab contains outright orders across all asset classes.
  • The Orders tab offers aggregation functionality allowing three aggregation buckets to view orders.
  • There is a split view once you click on each aggregation line showing further order details.
  • When the ‘active only’ button is checked (see green rectangle above), any closed orders are removed from the list.
  • There is a tool alert function in the form of a red bubble notification across all tabs. We are currently developing a customizable sound alert to complement this alert function.

MultiLeg Orders​

  • Users can be alerted of legged orders via an alert bubble on the MultiLeg Orders tab. The row will also be highlighted in the grid to allow users to quickly identify any working legged orders.
  • Orders that have been closed and were legged will not be highlighted.
  • The spread definition field provides better descriptive information.
  • Like the Orders tab, this tab has several combined columns to give it a cleaner look.

Auto Hedge States​

  • The Autohedge States tab is where users can see the status of all components of an autohedged order.
  • This tab will show users the amount of deltathathastradedforanautohedgegroup,aswellastheamountofdelta that has traded for an autohedge group, as well as the amount of deltathathastradedforanautohedgegroup,aswellastheamountofdelta that was hedged against it.
  • Users can be alerted of a legged autohedge via an alert bubble at the top of the screen on the Auto Hedge States tab. The row will also be highlighted in the grid.

Rejects​

The Rejects tab has an alert bubble and will show any orders that are rejected. The text field will show the reject reason.
