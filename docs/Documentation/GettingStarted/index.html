Getting Started

Welcome to SpiderRock Connect (aka Version 8 or V8), the most recent version of the SpiderRock trading platform running in next-generation data centers proximity-located to exchange matching engines. SpiderRock Connect includes several major new features, including a new ATS, a new API for sending and receiving live data objects called MLink and re-imagined tools that can be accessed via the web or desktop application. The popular SRSE/SQL API has also been upgraded, optimized and rebuilt while the classic Citrix-accessed tools are still available.

If a client firm wants to use the SpiderRock system, but is not yet a known entity, contact our Sales department to get onboarded.

Getting Credentials​

If a user does not already have SpiderRock login credentials, but their firm has been onboarded with SpiderRock, they can obtain credentials by contacting the SpiderRock Support Desk. If an existing firm relationship does not exist, a firm must first go through onboarding with SpiderRock's Client Support Desk.

Once a user has been created within the SpiderRock system, they will be issued depending on what the user needs to access:

For Next-Generation Tools (including access to the ATS):​

Users should contact a SpiderRock Support team member to create a SpiderRock ID and send an invitation to the user’s active email address. The user will receive this to their inbox where they will follow a link to complete their registration by saving a password, choosing their MFA method and confirming their MFA method. After completion of these steps, the user will be placed on the new Launcher page where they can launch any tools they have been configured to access.

For Legacy Tools (accessed through Citrix):​

A SpiderRock Support team member will generate a password for the user and send them via secure PDF.

For Data Users:​

A SpiderRock team member will generate an API key for the user and contact them to deliver the API key.

For SRSE:​

Users should contact a SpiderRock Support team member to either have a password generated for them or so they can tell them the password they would like. An existing SpiderRock ID will be needed.

Accessing SpiderRock via Tools​

Users can access tools in a few ways depending on which tools they need access to.

For Next-Generation Tools:​

Users can access the newest tools by logging in at https://www.spiderrockconnect.com or they can download the desktop version here. Note, users must log in with their next-generation SpiderRock ID along with associated MFA.

For Legacy Tools:​

Users can access legacy tools through Citrix by logging in at https://sms.spiderrock.net/vpn/index.html. Note, users must log in with the credentials sent via PDF from SpiderRock.

For SRSE:​

Users need a database client on their computer, an establish cross connect or VPN access. Using the credentials they have been provided, users can only connect via the public SRSE endpoints.

Accessing SpiderRock via APIs:​

Users can access the SpiderRock system in several programmatic ways using APIs.

Access via MLink via REST
Access via MLink via WebSocket
Access via SRSE
Access via FIX

Testing and Using the Sandbox​

In SpiderRock Connect, there is a test environment called the Sandbox. In this environment, there are pre-loaded datasets to help users test connection and applications in a predictable, safe environment with no risk of reaching the outside markets. To connect to the Sandbox, follow the instructions on our V8 Sandboxes here.
