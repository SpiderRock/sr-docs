Version: 8.6.5.4

V7 to V8 Migration

The V8 Migration process involves establishing connectivity to our new data centers in New Jersey and Chicago, migrating your scripts from SRSE V7 to SRSE V8 schemas, and working with our Client Support Desk to re-configure your production accounts, risk limits, drop-copies, and routes to executing brokers.

Our team will work closely with you through each step so that we can ensure a smooth and timely transition for your organization. For a list of important platform changes between V7 and V8, see the General Changes page.

warning

Migration to V8 must be completed by August 31st, 2026.

Establishing Connectivity in V8​

In the United States, SpiderRock Connect (V8) operates out of NY5 and DC3 data centers. Messages are bridged between the V7 and V8 environments to ensure real-time risk updates. This allows clients to trade in both systems concurrently and manage their migration in a phased approach.

Get Connected to V8​

  • Any existing cross-connects or VPN connectivity to V7 will have to be replicated in V8 data centers. Contact our Client Support Desk and Networking Team to get started.
  • To access European derivatives markets in V8, connectivity will need to be discussed with our Networking Team for Frankfurt (FR2) or possibly London (LD4) data centers.

For full endpoint details, IP addresses, and service URLs, see the Network Access page.

Trading in V8​

Via Citrix GUI Tools​

  • The Citrix tools in V8 are fully supported and function similarly to V7. If you currently access them through the cloud launcher, you can continue using the same launcher and login credentials — simply select the V8 tools from the menu.

    note

    In Europe, Citrix tools will not be supported.

Via our New IGNITE Tool​

  • V8 introduces IGNITE, a browser-based order entry and order management tool. For clients trading on European derivatives markets, IGNITE will be the designated tool since Citrix tools will not be supported. To download and get started with IGNITE, review the IGNITE Manual.
  • For a full overview of the other WebApp tools available in V8, review the Tool Suite section of the documentation. Login credentials will be different than the Citrix tools, our Client Support Desk can set users up with access.

Via SRSE​

  • Similar to V7, SRSE in V8 is not accessible over a public internet connection. A VPN or cross-connect to a SpiderRock V8 data center is required. To connect, target the following IP address:

    SRSE ProxyPort
    192.81.231.663700

  • For full setup instructions, see the SRSE Servers page.

Schema Differences​

Clients integrating with SRSE should review the schema differences between V7 and V8. For reference:

  • V7 to V8 Differences — Overview of structural differences
  • Messages — Per-message and field-level changes
  • Enums — Enum value and naming changes

See the SRSE API Changes section of the General Changes page for more details. Contact the Client Support Desk if you have any questions about the schema differences.

Account Configuration in V8​

At any given time, an account can only be active in one environment — either V7 or V8. This is controlled by an admin setting that determines the execution engine status per account.

There is no requirement to migrate all accounts at once. Clients can move a single account or a subset of accounts to V8 while keeping the rest in V7, facilitating a phased approach to migration. When a client is ready to begin migrating, the process is as follows:

  1. The client identifies the account(s) they would like to migrate to V8.
  2. SpiderRock configures the selected account(s) in V8 and the setup is approved by the Client Support Desk.
  3. SpiderRock schedules a meeting with the client and switches the account(s) from V7 to V8.
  4. The Support Desk reviews a client's routing counterparties to ensure everything is set up correctly in V8. Then the client tests connectivity and any other workflows to confirm that orders are routing and executing as expected.
  5. Once testing is confirmed successful, the account is flipped back to V7 to allow any open activity to clear.
  6. Once all the workflows have been verified, SpiderRock will work with the client on a full migration timeline to get the account moved to V8 permanently.

Risk Limits​

Several MAR Risk Limits have been modified in V8, which will require clients to complete a new Risk Form before trading in V8.

Third-Party Integrations​

If your firm uses any third-party integrations, coordinate with the Client Support Desk to review how connectivity and configuration should be moved to V8. SpiderRock will configure the equivalent connections in V8 and manage the switchover — no technical work is required on the client's part.

To facilitate the transition, clients should provide SpiderRock with a central point of contact (technical or business) at the third-party firm. This representative will help SpiderRock coordinate and prioritize the migration work with that vendor.
